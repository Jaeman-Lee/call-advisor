# PC 원격 음성 전사

## 목적

스마트폰은 통화 종료 감지, 16 kHz 변환, 결과 표시를 담당하고 같은 개인 네트워크의
PC가 `whisper.cpp` 추론을 담당합니다. 외부 음성 API는 사용하지 않으며 PC 연결이
실패하면 선택적으로 스마트폰의 `base` 모델로 복귀합니다.

```text
Android 녹음 → WAV 변환·VAD 측정 → SSH/SCP → PC small Whisper
                                              ↓
                  Markdown·CSV ← 전사문·성능 ←┘
```

## 실행 모드

| 설정 | 동작 |
|---|---|
| `local` | 스마트폰에서만 처리 |
| `remote` | PC를 사용하고 설정에 따라 local fallback |
| `auto` | PC를 우선 사용하고 실패 시 local fallback |

실제 호스트 주소, 사용자 이름과 키 경로는 Git에서 제외되는 `call-advisor.conf`에만
저장합니다. SSH 공개키 인증과 `BatchMode`를 사용하므로 비밀번호를 스크립트나 로그에
기록하지 않습니다.

## 실측

Ryzen 5 5600X 6코어, WSL2, CPU-only `small`, 6스레드와 Tailscale SSH를 사용했습니다.

| 입력 | 오디오 | PC Whisper | 왕복 전송 | 전체 처리 |
|---|---:|---:|---:|---:|
| 실제 한국어 통화 | 18.09초 | 6.51초 | 1.442초 | 10.318초 |
| 한국어 시험 녹음 | 35.92초 | 5.39초 | 약 1.6초 | 약 8.5초 |

전체 처리에는 스마트폰 FFmpeg 변환, VAD 측정, 전송, PC 추론 및 로컬 결과 저장이
포함됩니다. Codex가 비활성화된 시험이므로 선택적 텍스트 분석 시간은 제외했습니다.
콘텐츠와 발화 밀도에 따라 Whisper 디코딩 시간은 선형으로 증가하지 않을 수 있습니다.

31분 14초, 30.6 MB M4A의 지속 전송은 9.09초로 약 3.37 MB/s였습니다. 짧은 파일은
대역폭보다 SSH 연결 비용의 영향을 크게 받으므로 연결을 60초 동안 재사용합니다.

## PC 임시파일

처리 중 PC의 `incoming`과 `results` 디렉터리에 무작위 작업 ID가 붙은 WAV, 전사문,
성능 파일이 생성됩니다. 성공 또는 실패 후 삭제를 시도합니다. 비정상적인 전원 종료에
대비해 오래된 임시파일을 주기적으로 청소하는 운영 정책도 권장합니다.

원본 녹음 파일은 스마트폰에서 이동하거나 삭제하지 않습니다. 음성은 외부 API로
전달되지 않지만 지정 PC에는 복호화된 임시 WAV가 존재하므로 PC 디스크 암호화와 접근
통제를 적용해야 합니다.
